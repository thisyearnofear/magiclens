> **Goal:** Move from ~5/10 to ~9/10 across System Architecture, Product Design, UI/UX, and Intuitiveness — in time for the OKX X Cup submission.

**Core thesis:** The product has the right bones (dual-chain, SDK gen, AR editor, collaboration). The gap is that features don't compose into a journey. Every change below is about making them compose.

**Phases:**
- **Phase 1 (Days 1-2):** Architectural quick wins + UX essentials — low-risk, high-visibility fixes

# Phase 1: Architecture Tightening + UX Foundations

## Task 1.1: Extract a proper API client layer

**Objective:** Eliminate the `getAuthenticatedClient()` repetition pattern. One-time setup, clean call-sites.

**Files:**
- Create: `app/src/lib/sdk/client.ts`
- Modify: `app/src/lib/sdk/auth-client.ts` (deprecate)
- Modify: Every component that calls `getAuthenticatedClient()`

**Context:** Every API call today looks like:
```typescript
const response = await videoServiceGetVideos({
  client: getAuthenticatedClient(),
  body: { limit: 6, offset: 0 }
});
```

That's 3 extra lines per call-site for something that should be implicit.

**Step 1: Create the wrapped client**

`app/src/lib/sdk/client.ts`:
```typescript
import { createClient, createConfig } from '@hey-api/client-fetch';

let _client: ReturnType<typeof createClient> | null = null;

function getToken(): string | null {
  return localStorage.getItem('magiclens_token');
}

export function getApiClient() {
  const token = getToken();
  const headers: Record<string, string> = {};
  if (token) headers['Authorization'] = `Bearer ${token}`;

  if (!_client) {
    _client = createClient(
      createConfig({
        baseUrl: import.meta.env.VITE_API_BASE_URL || '',
        headers,
      })
    );
  } else if (token) {
    // Update auth header on the cached client; avoids re-creation on every call
    _client.setConfig({ headers: { ...headers } });
  }
  return _client;
}
```

**Step 2: Update the generated SDK to use the cached client**

The SDK is generated by `@hey-api/openapi-ts`. Rather than regenerating, we override the default client factory. In `app/src/lib/sdk/index.ts`:

```typescript
export { getApiClient } from './client';
```

Then update each service function to accept `client` as optional (defaulting to `getApiClient()`). Since the SDK is auto-generated, we patch the wrapper methods:

Actually — the cleanest path is to keep the SDK functions as-is but replace `getAuthenticatedClient` in every call-site. That means touching ~12 call-sites across 4 components. Each becomes:

```typescript
// Before:
const response = await videoServiceGetVideos({
  client: getAuthenticatedClient(),
  body: { limit: 6, offset: 0 }
});

// After:
const response = await videoServiceGetVideos({
  body: { limit: 6, offset: 0 }
});
```

This requires patching the SDK functions to inject `getApiClient()` as the default. In `app/src/lib/sdk/client.gen.ts`, each function signature has a `client` param — wrap with a default that comes from our singleton.

**Step 3: Update call-sites**

Files to modify:
- `app/src/components/Dashboard.tsx` — lines 43, 91, 163, 179
- `app/src/components/CollaborationWorkspace.tsx` — lines 43, 51
- `app/src/components/VideoGallery.tsx`
- `app/src/components/AssetLibrary.tsx`

Remove `client: getAuthenticatedClient()` from every call. The SDK handles it.

## Task 2.1: Create the "Remix" flow as the primary UX

**Objective:** Replace the generic "Upload Video → Browse Assets → Dashboard" with a single guided flow: "Pick a clip → Add AR → Mint → See leaderboard".

**Files:**
- Create: `app/src/components/RemixFlow.tsx`
- Create: `app/src/components/remix/ClipPicker.tsx`
- Create: `app/src/components/remix/ARWorkspace.tsx`
- Create: `app/src/components/remix/MintStep.tsx`
- Create: `app/src/components/remix/LeaderboardPreview.tsx`
- Modify: `app/src/components/Router.tsx` — add `/remix` route
- Modify: `app/src/components/Dashboard.tsx` — add "Create Remix" CTA

**The RemixFlow component:**

A multi-step wizard with clear numbered steps:
1. **Pick a clip** — User selects from recently uploaded World Cup clips or uploads a new one
2. **Add AR** — Full-screen editor with side panel showing available pack overlays (WorldCupPack items)
3. **Preview** — See the full remix playing with overlays
4. **Mint** — One-click mint on X Layer, auto-connecting EVM wallet if needed
5. **See results** — Show the remix on the daily leaderboard, with "top-3 could become Iconic Moment" callout

```typescript
// app/src/components/RemixFlow.tsx (structure)
export default function RemixFlow() {
  const [step, setStep] = useState(0);
  const [clip, setClip] = useState<Video | null>(null);
  const [overlays, setOverlays] = useState<OverlayData[]>([]);
  const [mintTx, setMintTx] = useState<string | null>(null);

  return (
    <div className="min-h-screen bg-gradient-to-br from-purple-900 via-blue-900 to-indigo-900">
      {/* Progress indicator: step 1 of 4 */}
      <StepProgress current={step} total={4} labels={['Clip', 'AR', 'Mint', 'Done']} />
      
      {step === 0 && <ClipPicker onSelect={(v) => { setClip(v); setStep(1); }} />}
      {step === 1 && <ARWorkspace clip={clip} overlays={overlays} onOverlaysChange={setOverlays} onNext={() => setStep(2)} />}
      {step === 2 && <RemixPreview clip={clip} overlays={overlays} onBack={() => setStep(1)} onMint={handleMint} />}
      {step === 3 && <MintConfirmation tx={mintTx} onViewLeaderboard={() => navigate('/leaderboard')} />}
    </div>
  );
}
```

## Task 3.1: Fix the guest → auth transition

**Objective:** No hard page reload when user connects a wallet. Smooth React transition.

**Files:**
- Modify: `app/src/components/Dashboard.tsx` — replace `window.location.href = '/'` with `navigate('/dashboard')`
- Modify: `app/src/components/WrongNetworkBanner.tsx` — verify smooth mount

**The problem:**
```typescript
// Guest mode "Connect Wallet" buttons do:
onClick={() => { window.location.href = '/'; }}
// This hard-reloads the page, losing all state.
```

**The fix:**
```typescript
onClick={() => navigate('/')}  // or wherever the connect flow lives
```

The landing page already renders the connect buttons. After auth, the `AuthProvider` updates `isConnected`, `SignedOut` stops rendering, `SignedIn` starts rendering, and the user sees the dashboard — all without a single page reload.

## Task 3.4: Add offline resilience to OverlayEditor

**Objective:** Editor interactions should never block on API calls.

**Files:**
- Modify: `app/src/components/OverlayEditor.tsx` — move API sync to background

**The problem (line 48-59):**
```typescript
const debouncedUpdateOverlay = useCallback(
  async (id: string, position: any, timing: any, layerOrder: number) => {
    await collaborationServiceUpdateOverlay({ body: {...} });
  }, []
);
```

Every canvas interaction triggers an awaited API call. If the network lags, the canvas freezes.

**The fix:**
```typescript
// Local state is the source of truth
const [overlays, setOverlays] = useState<OverlayData[]>(initialOverlays);

// Sync happens in background, never awaited
const syncOverlay = useRef(
  debounce(async (id: string, data: Partial<OverlayData>) => {
    collaborationServiceUpdateOverlay({ body: { overlay_id: id, ...data } })
      .catch(err => console.warn('Sync failed, will retry:', err));
  }, 1000)  // 1s debounce, long enough to batch rapid edits
).current;
```
